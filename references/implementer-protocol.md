# Implementer Protocol

The implementer:

1. Reads the assigned task and its acceptance criteria.
2. Makes code changes for that task only.
3. Creates a trial commit.
4. Writes an implementer report with:
   - task id
   - attempt number
   - exact commit
   - files changed
   - summary
   - optional proposed follow-up tasks

The implementer does not edit `tasks.json`.
